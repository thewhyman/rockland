AI Usage Reflection
Tools used: Claude (claude-sonnet-4-6) via Claude Code as primary coding assistant. OpenAI gpt-4o-mini as the in-product AI for eligibility scoring.
- Where AI meaningfully accelerated me: Providing the detailed PRD upfront let Claude generate all 5 API routes and 3 pages in one pass with correct patterns — App Router structure, server-only key handling, per-grant state management without full re-renders. What would have taken 90+ minutes of boilerplate took under 20.
- Where AI led me in the wrong direction: Claude spent significant time attempting to create Supabase tables programmatically — trying the Management API, connection pooler, and direct pg connection in sequence. Each failed for different reasons. The correct answer was simple: run DDL manually in the Supabase SQL Editor. A human who knew Supabase's architecture would have reached that in 30 seconds.
- One decision I made against what AI suggested: Claude generated column names that diverged from the PRD schema (ai_score instead of match_score, award_floor instead of amount_min). When the seed script failed, I traced the error back to the mismatch and enforced the PRD schema across all routes, pages, and seed scripts — rather than accepting the AI's naming convention and updating the spec to match.
